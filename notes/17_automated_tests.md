# 第 17 次学习：自动化测试

这一阶段学习的是：用代码检查代码。

之前我们修复了 `/plan-rename`，但如果每次都靠手动输入命令验证，很容易漏掉边界情况。自动化测试的价值是：把“我记得要检查什么”变成“程序每次都帮我检查”。

## 本次测试覆盖

新增测试文件：

```text
tests/test_plan_management.py
```

测试 1：重命名计划

- 旧计划 ID 会消失
- 新计划 ID 会出现
- active 指针会更新
- 原计划下的 mode 对话会迁移到新计划下

测试 2：删除计划

- 计划文件会删除
- active 指针会清空
- 该计划下所有 mode 对话会一起删除

## 运行方式

```powershell
python -m unittest discover -s tests
```

## 你需要理解

- 测试不是为了证明“现在能跑”，而是为了防止以后改坏。
- 测试应该尽量使用临时目录，避免污染真实 `data/`。
- 一个真实 bug 修复之后，最好补一个测试，让这个 bug 以后不容易回来。
