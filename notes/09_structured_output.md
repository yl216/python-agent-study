# 第 9 次学习：结构化输出

今天的目标是让模型不只是回答自然语言，而是输出程序能解析的 JSON。

## 为什么需要结构化输出

自然语言适合人读，但程序不好直接执行。

例如用户说：

```text
帮我计算 2 * (3 + 4)
```

程序需要知道：

- 要调用哪个工具？
- 工具参数是什么？
- 如果不该调用工具，要不要回到普通聊天？

所以我们让模型输出：

```json
{
  "intent": "calc",
  "argument": "2 * (3 + 4)"
}
```

## 本阶段新增能力

- 新增 `src/intent_parser.py`。
- 新增 `/intent <请求>` 命令。
- 模型负责把自然语言解析成 JSON。
- 程序负责校验 JSON，并决定是否调用工具。
- 如果模型连接失败，程序会使用本地规则做简单 fallback。

## 当前支持的 intent

- `calc`
- `time`
- `files`
- `note`
- `read`
- `summary`
- `todos`
- `save-summary`
- `chat`

## 关键边界

模型输出 JSON 不代表一定可信。

程序还要检查：

- JSON 是否能解析。
- 是否是对象。
- `intent` 是否在允许列表里。
- 工具执行是否安全。

## 本阶段验收

- `/intent 帮我计算 2 * (3 + 4)` 能解析成 `calc` 并调用计算器。
- `/intent 帮我总结 README.md` 能解析成 `summary` 并调用总结工具。
- 模型返回非法 JSON 时，程序能给出错误提示。
- 普通聊天类请求能解析为 `chat`，不调用工具。
- DeepSeek 临时连接失败时，常见工具请求仍可通过本地规则执行。
