# 第 3 次学习：多轮对话记忆

今天的目标是把“一问一答就结束”的程序，升级成可以持续聊天的 Agent。

## 本阶段新增能力

- 程序启动后进入 `while True` 循环。
- 用户可以连续输入问题。
- 输入 `exit` 或 `quit` 可以退出。
- 每轮用户问题会加入 `messages`。
- 每轮模型回答也会加入 `messages`。
- 程序只保留最近 N 轮对话，避免上下文无限增长。

## 核心概念

- `messages`：发送给聊天模型的上下文列表。
- `role=user`：用户说的话。
- `role=assistant`：模型回答过的话。
- `role=system`：对模型身份和行为的总体要求。
- 上下文：模型本身不会自动记住终端里的历史，程序必须把历史重新发给模型。

## 为什么要限制历史轮数

对话历史越长：

- 请求会越慢。
- 消耗的 token 越多。
- 成本可能更高。
- 无关历史可能干扰当前回答。

所以当前使用 `MAX_HISTORY_TURNS=5`，表示最多保留最近 5 轮用户和助手对话。

## 本阶段验收

- 程序不会回答一次就退出。
- 第二个问题可以引用第一个问题的内容。
- 输入 `exit` 能正常结束。
- `.env` 可以配置 `MAX_HISTORY_TURNS`。
