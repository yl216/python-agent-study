# 第 6 次学习：配置与密钥管理强化

今天的目标是把 `.env` 配置从“能读”升级成“能校验、能报清楚错误”。

## 为什么要做配置校验

真实项目里，很多错误不是代码逻辑错，而是配置写错：

- API Key 没填。
- URL 少了 `https://`。
- 温度写成了中文或字母。
- 历史轮数写成了 `0` 或很大的数。

如果没有校验，这些问题可能会在调用 API 时才暴露，而且错误信息不一定好懂。

## 本阶段新增能力

- 新增 `ConfigError` 表示配置错误。
- `DEEPSEEK_API_KEY` 必须填写，不能是示例占位值。
- `DEEPSEEK_BASE_URL` 必须以 `http://` 或 `https://` 开头。
- `DEEPSEEK_TEMPERATURE` 必须是 `0.0` 到 `2.0` 之间的数字。
- `MAX_HISTORY_TURNS` 必须是 `1` 到 `20` 之间的整数。
- `.env.example` 增加了注释，说明每个配置项的作用。

## 核心原则

- 真实密钥只放 `.env`，不要提交到 GitHub。
- 示例配置放 `.env.example`，可以提交。
- 配置错误要尽早失败，并给出人能看懂的提示。
- 默认值适合初学者，但关键密钥必须显式填写。

## 本阶段验收

- 没有 API Key 时，程序能提示缺少 `DEEPSEEK_API_KEY`。
- 温度写错时，程序能提示 `DEEPSEEK_TEMPERATURE` 必须是数字。
- 历史轮数写错时，程序能提示 `MAX_HISTORY_TURNS` 必须是整数或范围不对。
- `.env` 不会被 Git 提交。
